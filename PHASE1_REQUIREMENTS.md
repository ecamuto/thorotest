# Phase 1 ‚Äî Requirements & Coverage (v1.1.0)

Branch: `feat/requirements-coverage`
Goal: add a **Requirements** entity (features / stories / epics) linked many-to-many to
tests, with coverage metrics. Standalone value (manual + CSV/YAML import), and schema
pre-shaped for the later Jira integration (external_* fields mirror the defect plan).

Scope: **full** ‚Äî core (model/router/view/coverage/seed/tests/docs) **plus** CSV/YAML
requirements import and GraphQL `requirements` query. Full parity with the test entity.

Non-goals this phase: Jira sync itself, inbound webhooks. Those are Phase 2.

---

## 1. Data model (`backend/models.py`)

New `Requirement`:

| Column | Type | Notes |
|---|---|---|
| `id` | String(255) PK | e.g. `REQ-1001` (generated like `BUG-####`) |
| `title` | String(512) | required |
| `type` | String(32) | `feature` \| `story` \| `epic` (default `feature`) |
| `status` | String(32) | `draft` \| `active` \| `done` \| `deprecated` (default `active`) |
| `priority` | String(32) | reuse test priority vocab: low/med/high/critical |
| `description` | Text | nullable |
| `owner` | String(255) | nullable |
| `created_at` / `created_by` | String(64/255) | like Defect |
| `external_provider` | String(64) | nullable ‚Äî `jira` later |
| `external_key` | String(128) | nullable ‚Äî Jira issue key |
| `external_url` | String(512) | nullable |

Bridge table `requirement_tests` (many-to-many):
`requirement_id` FK ‚Üí requirements, `test_id` FK ‚Üí tests, unique pair.

Relationships:
- `Requirement.tests = relationship("Test", secondary=requirement_tests, back_populates="requirements")`
- `Test.requirements = relationship(..., back_populates="tests")` (add to existing Test)

**Also add `external_*` columns to `Defect`** now (same 3 fields) ‚Äî cheap, keeps both
entities symmetric for Phase 2 Jira, avoids a second migration.

## 3. Schemas (`backend/schemas.py`)

- `RequirementOut` (from_attributes) ‚Äî all cols + `test_ids: List[str]` + `coverage` block
  `{linked: int, passed: int, failed: int, untested: int, pass_rate: float}`.
- `RequirementCreate` ‚Äî title, type, status, priority, description, owner, test_ids.
- `RequirementUpdate` ‚Äî all optional, incl. `test_ids` (replace-set semantics).

## 4. Router (`backend/routers/requirements.py`)

Mirror `defects.py` structure exactly (roles, pagination, activity log):
- `GET /api/requirements` ‚Äî filters: `status`, `type`, `search`, `covered` (bool:
  has ‚â•1 linked test), pagination via `_pagination.paginate`.
- `GET /api/requirements/{id}` ‚Äî 404 pattern.
- `POST /api/requirements` ‚Äî id `REQ-####`, `log_activity(..., "created requirement", ...)`.
- `PATCH /api/requirements/{id}` ‚Äî incl. re-linking test_ids.
- `DELETE /api/requirements/{id}` ‚Äî admin only; cascade bridge rows.
- `POST /api/requirements/{id}/tests` / `DELETE /.../tests/{test_id}` ‚Äî link/unlink.

Coverage computed from linked tests' latest RunCase result (reuse the logic overview
uses for pass/fail; factor a small helper if needed).

Wire into `backend/main.py`: import + `include_router(requirements.router, prefix="/api")`.
